# Architecture overview

FlipFlip is a single-process Electron app with a React renderer and a TypeScript
Electron main process. Webpack builds two bundles into `dist/`.

## Main process (Electron)

- Entry: `src/main/main.ts`
- Window creation and lifecycle: `src/main/WindowManager.ts`
- Menus: `src/main/MainMenu.ts`
- IPC wiring: `src/main/IPCEvents.ts`

The main process creates BrowserWindows and loads the renderer HTML from
`dist/index.html` (generated by webpack). It also exposes a `--no-dev-tools`
flag to suppress auto-opening DevTools for the main window.

## Renderer (React)

- Entry: `src/renderer/renderer.tsx`
- Root HTML template: `src/renderer/index.html`
- UI components: `src/renderer/components/`
- Animations: `src/renderer/animations/`
- Global styles: `src/renderer/style.scss`
- Assets: `src/renderer/icons/`, `src/renderer/fonts/`

State and domain models live in `src/renderer/data/` (e.g., `Scene`, `LibrarySource`,
`Config`, `Playlist`). Actions and helpers live alongside those data models.

## Bundling

- `webpack.dev.js`: dev build, source maps for the renderer.
- `webpack.prod.js`: production build.
- Both configs output `dist/main.bundle.js` and `dist/renderer.bundle.js` and
  generate `dist/index.html` via `HtmlWebpackPlugin`.
- A compile-time `__VERSION__` constant is injected from `package.json`.

## Persistence (high-level)

Renderer state is persisted locally via `src/renderer/data/AppStorage.tsx`,
which reads/writes a JSON file under the app data directory. See
`agent-docs/base-docs/storage-and-backups.md` for details.
