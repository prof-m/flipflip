# User documentation map

The end-user manual lives in `docs/` (published at the project site). Use
these files when you change user-facing behavior or add UI features.

Key entry points:

- `docs/README.md`: top-level landing content
- `docs/_sidebar.md`: table of contents for the manual
- `docs/developers.md`: contributor setup and style guidance
- `docs/quickstart.md`: new-user setup steps
- `docs/config.md`: settings and API keys
- `docs/scenes.md`, `docs/options.md`, `docs/effects.md`: scene behavior
- `docs/library.md`, `docs/audio_library.md`, `docs/script_library.md`: libraries
- `docs/changelog.md`: release notes

If you add or change a feature, update the most relevant page and adjust
`docs/_sidebar.md` if navigation needs to change.
